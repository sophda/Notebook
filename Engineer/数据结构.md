# 哈希

### 1. 什么是哈希冲突？

哈希冲突是指**两个或更多不同的键，经过哈希函数计算后，得到了相同的哈希值，从而被映射到哈希表的同一个位置（桶）**。

**举个例子：**
假设我们有一个简单的哈希函数 `hash(key) = key % 10`，即用键除以10取余数作为哈希值。

- 现在插入 `key1 = 15`，`hash(15) = 5`
- 再插入 `key2 = 25`，`hash(25) = 5`

`15` 和 `25` 这两个不同的键被映射到了同一个位置 `5`，这就发生了**哈希冲突**。

**冲突是不可避免的：** 根据鸽巢原理，只要键的数量可能大于哈希表桶的数量，冲突就必然会发生。因此，一个好的哈希表设计**目标不是消除冲突，而是有效地处理冲突**。

------

### 2. 如何解决哈希冲突？

主要有两大类方法：**开放寻址法** 和 **链地址法**。

#### 方法一：开放寻址法

核心思想：**当发生冲突时，通过某种探测方法，在哈希表中寻找下一个空闲的桶，直到找到为止。**

所有的元素都存放在哈希表数组本身中，而不是像链地址法那样挂在链表或树上。这意味着哈希表必须足够大以容纳所有元素。

常见的探测序列方法：

1. **线性探测**
   - **方法：** 如果位置 `i` 被占用，就顺序检查 `i+1, i+2, i+3, ...` 直到找到一个空位。
   - **优点：** 实现简单。
   - **缺点：** 容易产生**聚集**。即出现一连串被占用的桶，导致后续插入和查找的效率下降（需要连续遍历很多次）。
2. **平方探测**
   - **方法：** 为了解决线性探测的聚集问题，使用一个平方函数进行探测。如果位置 `i` 被占用，接下来检查 `i + 1², i - 1², i + 2², i - 2², i + 3², ...`。
   - **优点：** 缓解了线性探测的聚集问题。
   - **缺点：** 不一定能探测到哈希表的所有位置。
3. **双重散列**
   - **方法：** 使用**第二个哈希函数**来计算探测步长。如果位置 `i` 被占用，则计算 `i + 1 * hash2(key), i + 2 * hash2(key), i + 3 * hash2(key), ...`。
   - **优点：** 提供了最好的探测序列，不同的键有不同的步长，能有效减少聚集。
   - **缺点：** 计算量稍大。

**开放寻址法的优缺点：**

- **优点：** 所有数据都在一个数组中，易于序列化；利用CPU缓存局部性更好（数据更连续）。
- **缺点：** 对哈希函数和负载因子非常敏感；当表快满时，插入和查找性能会急剧下降；删除操作复杂（需要特殊标记，不能直接置空，否则会中断后续的查找）。

#### 方法二：链地址法

核心思想：**哈希表的每个桶不是一个直接存储数据的单元，而是一个指针，指向一个链表（或其他容器）。所有映射到同一个桶的键值对都存储在这个链表中。**

- **查找：** 先计算哈希值找到桶，然后遍历该桶对应的链表，使用 `equals()` 方法比较键，直到找到匹配的节点。
- **插入：** 先查找，如果存在则更新值；如果不存在，则将新节点插入到链表尾部（或头部）。
- **删除：** 先查找，找到后从链表中移除该节点。

**链地址法的优化：**
在JDK 8的HashMap中，当链表长度过长时（默认 >= 8），会将链表转换为**红黑树**，从而将最差情况下的查找时间从 `O(n)` 优化到 `O(log n)`。当桶中的节点数因删除而减少时（默认 <= 6），红黑树又会退化成链表，以节省空间。

**链地址法的优缺点：**

- 实现简单，有效。
- 对负载因子容忍度高（即使负载因子 > 1，即元素数大于桶数，也能正常工作）。
- 删除操作简单，直接操作链表即可。



缺点：

- 节点需要额外的指针空间（存储链表的下一个节点或树的左右子节点），内存开销稍大。
- 链表过长时（未优化前）性能会下降。缓存局部性不如开放寻址法（节点在内存中不连续）。

------

### 总结与对比

| 特性           | 开放寻址法                     | 链地址法                             |
| :------------- | :----------------------------- | :----------------------------------- |
| **核心思想**   | 在数组内寻找下一个空位         | 每个桶指向一个链表/树                |
| **数据存储**   | 全部存储在数组中               | 节点分散在数组和链表/树中            |
| **内存占用**   | 无额外指针开销，但要求数组更大 | 有额外指针开销，但数组可以更小       |
| **性能影响**   | 聚集现象会导致性能下降         | 链表过长会导致性能下降（可用树优化） |
| **删除操作**   | 复杂，需特殊标记               | 简单，直接操作链表                   |
| **缓存友好性** | **好**（数据连续）             | 较差（节点分散）                     |
| **实现难度**   | 相对复杂                       | 简单直观                             |

## 二叉树遍历

leetcode模式--二叉树前序遍历，使用递归的方式：

```c++
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode() : val(0), left(nullptr), right(nullptr) {}
 *     TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
 *     TreeNode(int x, TreeNode *left, TreeNode *right) : val(x), left(left), right(right) {}
 * };
 */
class Solution {
public:
    void preorder(TreeNode* node, vector<int >& vec) {
        if(node == nullptr) return;
        vec.push_back(node->val);
        preorder(node->left, vec);
        preorder(node->right, vec);

    }

    vector<int> preorderTraversal(TreeNode* root) {
        vector<int > temp;
        temp.reserve(200);

        preorder(root, temp);

        return temp;
        
    
    }
};
```



## 最大堆

最大堆是一种特殊的**完全二叉树**数据结构，它满足以下关键性质：

1. **堆序性质**：每个节点的值都**大于或等于**其子节点的值，即：对于任意节点 i，`value(i) ≥ value(left_child(i))` 且 `value(i) ≥ value(right_child(i))`

2. **结构性质**：是一棵**完全二叉树**（所有层级都被填满，最后一层从左到右填充）

实例：

```
        100
       /    \
      90     80
     /  \    / \
    70  60  50  40
   /
  30
```

## 生成树

**生成树**是一个连通图G的一个极小的连通子图，包含图G的所有顶点，但是只有`n-1`条边，并且是联通的。

生成树可以由遍历过程中的经过的边组成：

- 深度优先遍历得到的数称为**深度优先生成树**
- 广度优先遍历得到的树称为**广度优先生成树**

---

- 极大连通子图：最多顶点，最多边的连通图
- 极小连通子图：最多顶点，最少边的连通图

## 最小生成树

1.定义：在一个连通网的所有生成树中，各边的代价之和最小的那棵生成树称为该连通网的最小代价生成树简称为最小生成树（MST）

例如：要在n个城市间建立交通网，要考虑的问题如何在保证n点连通的前题下最节省经费?

求解:连通6个城市且代价最小的交通线路? 

2.重要性质

> 连通网N=（V,E）
>
> - V表示图中的所有顶点的集合
> - E表示图中所有边的集合

设N=(V, E) 是一连通网，U 是顶点集V的一个非空子集。若(u , v) 是一条具有最小权值的边，其中u∈U，v∈V-U，则存在一棵包含边 (u , v) 的最小生成树。 

---

- prim算法：初始化一个顶点，然后选择这个顶点中权值最小的边，然后将新的顶点加入，然后不断添加这个新的顶点集合的权值最小的边
